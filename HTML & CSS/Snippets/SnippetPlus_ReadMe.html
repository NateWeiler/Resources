HOW TO INSTALL

1. Copy SnippetPlus.dll into Notepad++ plugin folder
2. (OBSOLETE if you are using the latest version) Copy SnippetPlus folder into Notepad++ plugin folder as well. SnippetPlus folder contains a file SnippetPlusPlus.dll
3. OPTIONAL: Copy SnippetPlus.xml into Notepad++ config folder.
If you don't, program will create a default SnippetPlus.xml file for you. (But why not use the one already created by some hardworker, ME :D)

HOW TO USE

You have four different ways to use SnippetPlus for Snippets.
1. By using snippet name and shortcut.
2. By using the dockable window. (Click funny looking SN Icon or Select Plugins --> SnippetPlus --> Dock SnippetPlus).
3. By using the floating window. (Select Plugins --> SnippetPlus --> Show GUI Editor).
4. By using the right click


You have three different ways to use SnippetPlus for SurroundWith.
5. By using the dockable window. (Check the checkbox "Use Surround With")
6. By using the floating window. (Check the checkbox "Use Surround With")
7. By using the right click.
Important changes into version 1.4:
Only one DLL is needed from this version
This version recreated from scratch. Let me know asap if anything behaving differently.

Important changes into version 1.2:
Add the folowing to your SnippetPlus.xml file just before the </SnippetPlus>
<SnippetPlusOptions>
<Option UseSurroundWithOnLoadForEditor="False" UseSurroundWithOnLoadForDockable="True" MaxItemsInAutoComplete="10" />
</SnippetPlusOptions>
or let the plugin create a new file for you. Will not be created into the existing file.

Important changes into version 1.1:
If you are using old version of SnippetPlus.xml file, then please change the following:
From: <Snippet surroundWith="someName">
To: <Snippet surroundWith="someName" indent="">
and add enabled attribute even for SurroundWith's Lang element.
If you don't, it should work well as I have put some check, but in the future I will make it necessary.

If you are confused, rename your SnippetPlus.xml, restart Notepad++ and let the plugin make one for you to get the idea.

DETAILED EXPLAINATION ON HOW TO USE

================================================================================
ad 1. The fastest way (if you have good memory though :)
- Write the snippet name and then press the shortcut. (Default is: CTRL + SHIFT + ENTER) but you are free to change it to suit your needs.
- If you only remember part of the snippet name, then after pressing CTRL + SHIFT + ENTER, it will show you all the snippets matching what you typed.
- Pressing CTRL + SHIFT + ENTER on empty space will show all the snippets for the active document language (or for the GENERAL language).

NOTE:
Before you ask Why this shortcut:
Well, Notepad++ itself uses so many shortcuts.
Then depending on how many plugins you have already installed, your Notepad++ will be using more shortcuts and as so it is not an easy task to find a free shortcut.
So I choosed one but you can easily change it by:
Settings --> Shortcut Mapper --> Choose Tab "Plugin Commands" --> Find the text "Insert Code Snippet" and modify the shortcut.


ad 2. By using dockable window.
- Click on the funny looking SN icon on the toolbar OR select Plugins --> SnippetPlus --> Dock SnippetPlus.
- After that you will have a docked window.
- Combobox shows loaded languages and below are all the snippets for that language. Changing language will load snippets for that language.
- Select a snippet and click on Insert button OR just double click on the snippet name as it is faster and makes the Insert button actually needless. I may delete it in the future.
- If you changed the SnippetPlus.xml file, Reload button will load the changes.
- Filter Textbox will show only those snippets which will contain the text from this filter. Useful if you have a lot of snippets.
- Changing the file in Notepad++ or changing the language type for the active document will automatically show snippets for that language.


ad 3. By using the floating window.
- Select Plugins --> SnippetPlus --> Show GUI Editor.
- It has all the features from point 2. plus it works as an editor for Snippets and SurroundWith.


ad 4. By using the right click.
- Right click option works assuming you have the following in your contextMenu.xml

- Put cursor on a word, right click and choose Insert Code Snippet. The rest is same as for point 1.


ad 5. By using dockable window.
- Click on the funny looking SN icon on the toolbar OR select Plugins --> SnippetPlus --> Dock SnippetPlus.
- Check the checkbox "Use Surround With" to activate.
- Select some text (not necessary), then double click the snippet name or select snippet and click insert.


ad 6. By using the floating window.
- Select Plugins --> SnippetPlus --> Show GUI Editor.
- Check the checkbox "Use Surround With" to activate.
- Select some text (not necessary), then double click the snippet name or select snippet and click insert.


ad 7. SurroundWith Functionality
- Right click option works assuming you have the following in your contextMenu.xml

- Select some text (not necessary), right click and then choose SurroundWith.
- It will popup a list of surrondwith commands for the given doc's language. (Or from GENERAL)
- choose the name and the selected text will be surrounded with it.

EDITOR EXPLAINATION

================================================================================
Button Rename is disabled:
If a snippet exists with the name of the text you entered into the textbox then there is nothing to rename. Else will rename to the selected snippet with what you type in the textbox.


Button Modify:
If a snippet exists with the name of the text you entered into the textbox then you can only modify.
If the name does not exist, then the text changes to "Add New", telling you that you will be creating a new snippet.

THINGS TO KNOW

================================================================================
1. Languages with 0 snippets will not be loaded. (Can be changed in the future versions if users would prefer that way)
2. Languages with parameter enabled="false" will not be loaded.
3. For shortcut use, langName parameter in SnippetPlus.xml file should be used as:
ADA, ASM, ASP, AU3, BASH, BATCH, C , CAML, CMAKE, COBOL, CPP, CS , CSS, D , DIFF, FLASH, FORTRAN, GUI4CLI, HASKELL HTML, INI, INNO, JAVA, JS , KIX, LISP, LUA, MAKEFILE, MATLAB, NFO, NSIS, OBJC, PASCAL, PERL, PHP, POWERSHELL, PROPS, PS , PYTHON, R , RC , RUBY, SCHEME, SEARCHRESULT, SMALLTALK, SQL, TCL, TEX, TXT, USER, VB , VERILOG, VHDL, XML, YAML so Notepad++ will know from which language to take the snippet.
Except these langName parameters, there is one special langName parameter GENERAL which will also be used when using shortcut.

How GENERAL works:
If you type a snippet name and press the shortcut, it will look for a snippet in the active document language.
If not found there, then it will try to find it in GENERAL language.


Example:
Lets say you are working with C# language selected in Notepad++. You type "if" and press shortcut.
It will try to find the snippet with name "if" in the snippets for language with parameter langName="CS". If it does not find it there, it will try to find it in GENERAL.
The purpose of this GENERAL langName was to probably reduce some writing. A typical example to use it can be "if statement" as I believe it is same in all C type languages. So instead of typing it in CS, C++, C, you can simply write it once in GENERAL. Or you may use GENERAL for your address, company name and so on

For GUI: langName parameter can be any name you wish.
4. The plugin has been created using C#, Linq, so you will need framework 3.5. (Sorry for inconvenience)
5. Parameter ignoreCase="TRUE" means Snippet name CLASS and class are treated the same way, otherwise not. But note that you CANNOT have CLASS and class in one language.
The idea behind implementing ignoreCase was that you can write a snippet name as SqlConnection for better readability and you can use it as sqlconnection as a shortcut.
6. I am using my plugin on daily bases without a problem but it has only been tested on my comp (XP) and friends comp (Win 7).
